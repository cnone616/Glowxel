## 七、跨模块联动分析(重点)

### 7.1 设备连接链路

```
UniApp overview -> ConnectModal输入IP -> deviceStore.connect(ip)
-> WebSocket ws://<ip>/ws -> ESP32 WS_EVT_CONNECT
-> 返回 {status:"connected", mode:"canvas/animation"}
-> deviceStore.deviceMode同步 -> 页面UI跟随
```

**[已完整]**

### 7.2 像素画编辑->板子显示

```
editor PixelCanvas触摸 -> 像素Map -> 二进制(x,y,r,g,b) x N
-> WebSocket.send(ArrayBuffer) -> ESP32 handleBinaryData
-> canvasBuffer + drawPixel -> 500ms后保存Preferences
```

**[已完整]**

### 7.3 GIF动画完整链路(最复杂)

```
选GIF -> _handleGifFile -> readFile -> _saveGifToLocal -> _parseAndInitGif
上传 -> animation_begin -> 逐帧frame_begin+binary -> animation_end
板子重启 -> loadClockConfig恢复dispMode -> loadAnimation -> 自动播放
UniApp重开 -> loadConfig检测hasGif -> _restoreGifFromLocal -> 完整恢复
```

**[已完整]** -- 全链路已打通

### 7.4 联动风险点(均已修复)

| 场景                        | 风险                 | 状态              |
| --------------------------- | -------------------- | ----------------- |
| 动态->静态->动态切clockMode | GIF状态不一致        | [已修复] M3       |
| WebSocket断线快速重连       | 无指数退避           | [已修复] M2       |
| ESP32只支持单客户端         | 第二个连接挤掉第一个 | [低风险] 设计如此 |
