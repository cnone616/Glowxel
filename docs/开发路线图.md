# Glowxel 开发路线图

更新日期：2026-04-24

说明：本路线图中的 `Glowxel` 指仓库 / 工程代号；用户可见命名统一按 `光格像素工坊 / Glowxel PixelBoard / RenLight` 理解。

## 1. 文档定位

这份文档只保留当前仍然有效的主线，不再保留已经过时的阶段性任务拆分。

目标只有 3 个：

1. 说明项目现在正在做什么
2. 说明每个主模块后续优先级
3. 避免继续新增重复规划文档

## 2. 当前项目阶段

本仓库（工程代号 `Glowxel`）当前已经不是单一像素时钟项目，而是一个由多端协同组成的像素内容与设备系统。

当前主线分为 4 条：

1. `website`
   - 承接官网浏览壳、应用主端壳、在线创作、社区展示、图纸工作台，以及“基础设备连接 + 关键桌面模式”
2. `uniapp`
   - 承接用户主操作链路、设备控制、编辑与同步
3. `esp32-firmware`
   - 承接真实显示、主题模式、效果模式和设备网页
4. `server + admin`
   - 承接数据、发布、管理与交付支撑

## 3. 当前优先级

### 3.1 第一优先级：设备体验收口

核心目标：

1. 保持 `WS`、时间同步、模式切换、素材发送稳定
2. 板载效果与前端预览尽量一致
3. 已接入主题继续做完整复刻，而不是只做“可切换”
4. 避免为了扩功能破坏原先稳定链路

当前重点：

1. `website` 设备链只保留 `device-control / ble-config / device-params / canvas-editor`
2. `website` 设备正式可见模式收缩为“连接设备 + 画板模式”；拼豆工作台保留为桌面创作主链
3. 时钟、宠物、屏保、隐藏场景页不再作为 `website` 当前正式能力推进，统一回到 `uniapp` 主设备链
4. `weather / countdown / stopwatch / notification` 本轮继续排除，不再回灌到网站正式功能

### 3.2 第二优先级：图纸工作台

核心目标：

1. 以 `website` 为完整版工作台主阵地
2. 不把图纸能力缩成只剩基础图片转像素
3. 核心算法尽量和 `uniapp` 共用

当前重点：

1. `workspace -> create -> editor` 保留为当前网站正式创作链
2. `PatternWorkbench` 保留为高级工作台，继续承接拼豆图纸和辅助校对
3. `overview / assist / publish-project / profile / my-works / my-favorites / achievements / cloud-sync / settings` 继续保留为网站正式非设备链路

### 3.3 第三优先级：视觉系统统一

核心目标：

1. 图标、封面、功能入口风格统一
2. 减少页面间风格割裂
3. 保持白色极简主基调，不做无意义重设计

当前重点：

1. `website` 正式页统一落到 `/design-system` 的 `glx-page-shell / glx-topbar / glx-panel-card / toast / blocking loader` 组件体系
2. `Templates / Challenges / ChallengeDetail / ArtworkDetail / UserProfile / FollowList` 已统一回到 `glx` 页面族
3. `website` 不再追求 `uniapp` 全量模式等价，优先把桌面创作与基础设备链做强

### 3.4 第四优先级：账号体系收口

核心目标：

1. `website` 当前先不暴露普通用户账号入口
2. `uniapp` 保留微信登录，并在登录后绑定手机号
3. `server` 负责账号合并、统一用户会话和记录归并

当前重点：

1. 网站端登录、个人中心、作品管理、发布和互动页面继续保留，避免网站主端能力残缺
2. 已冻结网站会话 key：`auth_token / currentUser / loginTime`
3. 当前仍被阻塞在“手机号注册/登录/绑定/合并字段合同未冻结”
4. 在字段合同确认前，不继续发明数据库列、接口参数名或兼容字段

### 3.5 第五优先级：仓库治理

核心目标：

1. 清理临时脚本、一次性预览页和过时文档
2. 收敛项目说明入口
3. 降低后来维护时的理解成本

当前重点：

1. 清理根目录临时预览工具
2. 只保留当前有效文档
3. 减少“阶段计划 + 进度文档 + 审计文档”并存造成的重复口径

## 4. 当前收尾进度

### `website`

已完成：

1. 官网浏览壳与应用主端壳分流已经落地
2. 设备控制页已收口为“基础设备链 + 关键桌面模式”，不再把所有设备模式继续铺到 `website`
3. `canvas-editor` 保留为当前唯一正式设备模式页，拼豆工作台保留为桌面创作强入口
4. `Templates / Challenges / ChallengeDetail / ArtworkDetail / UserProfile / FollowList` 已切到 `glx` 页面族
5. `ble-config` 已固定为“设备热点 -> `192.168.4.1` -> 写入 2.4GHz WiFi -> 返回设备控制”主链说明
6. `device-params` 已改成未连接即返回设备控制，并沿用固件现有字段合同
7. 登录、个人中心、项目总览、辅助处理、发布页、作品管理、收藏页和 QA 路由都继续保留在网站正式页面集合中
8. 会话存储已开始统一到 `auth_token / currentUser / loginTime`

未完成且仍阻塞：

1. 普通用户手机号注册/登录接口与数据库字段合同
2. 微信绑定手机号后的账号合并服务端事务链
3. `server + uniapp` 认证收口需要先确认新增字段映射表，再继续编码
