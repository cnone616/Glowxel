# Glowxel 开发路线图

更新日期：2026-04-07

## 1. 文档定位

这份文档只保留当前仍然有效的主线，不再保留已经过时的阶段性任务拆分。

目标只有 3 个：

1. 说明项目现在正在做什么
2. 说明每个主模块后续优先级
3. 避免继续新增重复规划文档

## 2. 当前项目阶段

Glowxel 当前已经不是单一像素时钟项目，而是一个由多端协同组成的像素内容与设备系统。

当前主线分为 4 条：

1. `website`
   - 承接在线创作、社区展示和图纸工作台
2. `uniapp`
   - 承接用户主操作链路、设备控制、编辑与同步
3. `esp32-firmware`
   - 承接真实显示、主题模式、效果模式和设备网页
4. `server + admin`
   - 承接数据、发布、管理与交付支撑

## 3. 当前优先级

### 3.1 第一优先级：设备体验收口

核心目标：

1. 保持 `WS`、时间同步、模式切换、素材发送稳定
2. 板载效果与前端预览尽量一致
3. 已接入主题继续做完整复刻，而不是只做“可切换”
4. 避免为了扩功能破坏原先稳定链路

当前重点：

1. `theme` 模式继续补齐成熟主题表现
2. 宠物眼睛、氛围效果、方块时钟继续提升真实观感
3. 画板、素材、效果编辑器保持和设备显示链路一致

### 3.2 第二优先级：图纸工作台

核心目标：

1. 以 `website` 为完整版工作台主阵地
2. 不把图纸能力缩成只剩基础图片转像素
3. 核心算法尽量和 `uniapp` 共用

当前重点：

1. 图纸生成
2. 图纸导入
3. 智能校对
4. 差异比对
5. 精修与导出

### 3.3 第三优先级：视觉系统统一

核心目标：

1. 图标、封面、功能入口风格统一
2. 减少页面间风格割裂
3. 保持白色极简主基调，不做无意义重设计

当前重点：

1. `uniapp` 功能入口图标替换
2. 成就、封面、缩略图资源整理
3. 设计资源产物格式与实际工程接入方式统一

### 3.4 第四优先级：仓库治理

核心目标：

1. 清理临时脚本、一次性预览页和过时文档
2. 收敛项目说明入口
3. 降低后来维护时的理解成本

当前重点：

1. 清理根目录临时预览工具
2. 只保留当前有效文档
3. 减少“阶段计划 + 进度文档 + 审计文档”并存造成的重复口径
