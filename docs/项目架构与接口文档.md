# 项目架构与接口文档

更新日期：2026-04-07

## 一、模块结构

仓库当前分为 5 个主模块：

- `website/`
  - 官网、社区、在线创作、图纸工作台、网页设备入口
- `admin/`
  - 运营后台
- `server/`
  - API 服务
- `uniapp/`
  - 小程序 / H5 / App 客户端，承接主设备控制链
- `esp32-firmware/`
  - ESP32 设备固件与设备网页

补充入口：

- 页面分类总入口见 [文档与页面索引.md](/Users/aflylong/Desktop/project/Glowxel/docs/文档与页面索引.md)
- 当前开发主线见 [开发路线图.md](/Users/aflylong/Desktop/project/Glowxel/docs/开发路线图.md)
- `website` 路由入口见 `website/src/router/index.js`
- `admin` 路由入口见 `admin/src/router/index.js`
- `uniapp` 页面目录位于 `uniapp/pages/`

## 二、当前职责边界

### `website`

负责：

- 官网展示
- 社区浏览
- 在线创作入口
- 图纸工作台
- Web 设备连接
- Web 时钟编辑
- 工作台和个人相关页面

当前不负责：

- 设备热点配网
- 高复杂度设备编辑流
- 高频设备主控体验

### `admin`

负责：

- 用户、作品、评论、模板、挑战、活动管理
- 固件版本管理
- 基础运营数据看板

### `uniapp`

负责：

- 像素创作
- 社区互动
- 热点配网引导
- 设备控制
- 时钟编辑
- 主题模式
- 天气 / 倒计时 / 秒表 / 通知提醒功能编辑
- 宠物眼睛模式
- 与设备实时同步

### `server`

负责：

- 用户与管理员鉴权
- 项目与作品数据
- 社区互动数据
- 固件版本接口
- 文件上传

### `esp32-firmware`

负责：

- 设备配网
- 设备网页
- WebSocket 命令接收
- 像素画布显示
- 静态 / 动态时钟显示
- 主题渲染
- 原生业务效果
- 动画播放
- OTA 更新

## 四、设备通信

### HTTP

设备 HTTP 接口由固件侧提供，当前主链仍包含：

- `/status`
- `/test`
- `/upload`
- `/brightness`
- `/text`
- `/clear-wifi`
- `/themes`
- `/effects`
- `/theme-frame.bin`
- `/live-frame.bin`
- `/theme-frame.txt`
- `/live-frame.txt`

### WebSocket

设备 WebSocket 路径：

- `ws://<device-ip>/ws`

当前设备主模式：

- `clock`
- `animation`
- `canvas`
- `theme`
- `transferring`

当前业务模式标签还包括：

- `text_display`
- `breath_effect`
- `rhythm_effect`
- `ambient_effect`
- `tetris`
- `weather`
- `countdown`
- `stopwatch`
- `notification`
- `eyes`

当前命令已按类别拆分：

- 基础信息
- 模式切换
- 时钟配置
- 主题配置
- 动画传输
- 画布与像素显示
- 文本与原生效果
- 眼睛交互
- OTA 相关
