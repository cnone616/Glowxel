# 当前仓库状态与收口说明

更新日期：2026-03-26

## 一、仓库概况

当前仓库包含 5 个主要端：

- `website/`：官网与在线创作站
- `admin/`：后台管理
- `server/`：后端 API
- `uniapp/`：客户端与设备控制
- `esp32-firmware/`：ESP32 固件

## 二、当前已落地结果

### 已完成的高优先级治理

- 仓库敏感信息已清理
- `server` 管理员密码方案已从 `sha256` 迁移到 `bcrypt`
- `server` 已补最小测试底线
- 已新增基础 CI 工作流
- `server` 已新增数据库迁移机制
- 多端接口地址已收口到统一入口
- `website` 时钟页面已明确为说明页，不再伪装为完整 Web 时钟编辑器
- `esp32-firmware` 的 WebSocket 命令处理已完成第一轮职责拆分

### 当前已验证模块

- `website`
  - `npm test` 通过
- `admin`
  - `npm test` 通过
- `server`
  - `npm test` 通过
- `esp32-firmware`
  - `platformio run` 通过

## 三、固件当前真实状态

### 配网方式

- 已从旧 AP 配网页方案切换为 BLE 配网主链
- BLE 设备名：`RenLight-Setup`
- 小程序页面：`pages/ble-config/ble-config.vue`
- 为兼容小程序“重置网络”操作，固件保留 `GET /clear-wifi`

### 设备模式

当前模式为：

- `clock`
- `animation`
- `canvas`
- `transferring`

### 本轮已收口的固件问题

- 静态时钟背景缓存缺失，已补齐
- `diff` 动画帧被误当全帧清屏，已修正
- 小程序仍依赖的 `/clear-wifi`，已恢复
- `websocket_handler.cpp` 的 JSON 命令处理已拆到独立命令分组文件
- 固件当前可直接用 `platformio run` 编译
