## 📋 数据库设计

### 1️⃣ 用户表 users
```javascript
{
  _id: userId,
  _openid: "",           // 微信云开发自动添加
  nickname: "",
  avatar: "",
  bio: "",
  createdAt: Date,
  followersCount: 0,     // 关注者数量（预留）
  followingCount: 0,     // 关注数量（预留）
  likeReceived: 0,       // 收到的点赞数
  publishCount: 0,       // 发布作品数
  role: "user"           // user | admin
}
```

### 2️⃣ 作品表 posts（最重要）
```javascript
{
  _id: postId,
  _openid: "",           // 作者标识
  userId: "",            // 作者ID
  title: "",
  coverUrl: "",          // 封面图URL（云存储）
  previewUrl: "",        // 预览图URL（可选）
  
  // 画布信息
  width: 104,
  height: 104,
  boardCount: 4,
  
  // 像素数据（压缩后）
  pixelData: {},         // JSON字符串或压缩格式
  
  // 统计信息
  colorCount: 12,
  tags: ["动物", "像素风"],
  
  // 状态
  status: "normal",      // normal | banned | hidden
  
  // 互动数据
  likeCount: 0,
  commentCount: 0,
  collectCount: 0,
  
  createdAt: Date,
  updatedAt: Date
}
```

**⚠️ 关键点：**
- pixelData 一定要压缩
- 不能直接存 Map
- 可以转成：JSON 字符串 或 RLE 压缩 或 二维数组压缩

### 3️⃣ 点赞表 likes
```javascript
{
  _id: likeId,
  _openid: "",           // 点赞用户
  userId: "",
  postId: "",
  createdAt: Date
}
```

**好处：**
- 可取消点赞
- 可统计
- 可防止重复点赞

### 4️⃣ 收藏表 collections
```javascript
{
  _id: collectionId,
  _openid: "",           // 收藏用户
  userId: "",
  postId: "",
  createdAt: Date
}
```

### 5️⃣ 评论表 comments
```javascript
{
  _id: commentId,
  _openid: "",           // 评论用户
  postId: "",
  userId: "",
  content: "",
  createdAt: Date,
  status: "normal"       // normal | banned
}
```

## 📝 技术要点

### 图片存储
- ❌ 不要把图片 base64 存数据库
- ✅ 生成预览图 → 上传云存储 → 存图片 URL

### 发布流程
1. 生成封面图
2. 压缩 pixelData
3. 上传封面图
4. 写入数据库

### 审核机制
- 作品表里加 `status: normal | banned | hidden`
- 后台可以：下架、屏蔽、删除
- 前期可以人工审核

### 数据压缩
像素图数据本身非常小：
- 104×104 = 10816 格
- 压缩成 `{"0,1": "#000000", "0,2": "#ffffff"}`
- 大约几十 KB
