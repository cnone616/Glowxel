# 设备工作台新功能规划（uniapp + 板载程序）

## 1. 产品目标

把当前“设备控制 + 时钟编辑”升级为统一的`设备工作台`，主功能并列，不再强依赖“时钟”概念。

### 1.1 主功能清单（冻结）
1. `静态内容`
2. `动态内容`（含“显示时间”开关，关闭即自定义动画）
3. `文本展示`
4. `呼吸灯`
5. `律动`
6. `画板模式（拼豆）`
7. `俄罗斯方块`

## 2. 交互规则（冻结）

1. 点击主功能按钮：仅切换前端“待编辑功能”，不立即切设备模式。
2. 用户点击`保存并应用`后：才下发对应模式和参数到设备。
3. 拼豆模式是独立主功能，进入拼豆后可实时显示，退出自动恢复上一次业务模式。
4. 不从设备状态反向覆盖前端主功能选择（前端主导）。
5. 所有可配置开关（速度/循环/方向/强度/颜色等）由`uniapp`提供并下发，用户不需要接触板载程序。
6. 文本展示必须支持中文，不做“仅英文数字”的过渡方案。

## 3. 页面结构规划

## 3.1 设备工作台布局
1. 顶部：连接状态、设备 IP、亮度、清屏。
2. 功能区：7 个主功能按钮（单选）。
3. 参数区：按当前主功能显示对应参数卡片。
4. 操作区：`保存并应用`、`保存预设`、`读取预设`。

## 3.2 各主功能参数草案
1. 静态内容：图片、静态文本、静态时钟开关、位置/颜色。
2. 动态内容：GIF/帧动画、时间叠加开关、速度、循环。
3. 文本展示：文本内容、字体、速度、方向、循环。
4. 呼吸灯：颜色、最小亮度、最大亮度、周期、波形。
5. 律动：BPM、强度、颜色组、模式（脉冲/渐变/跳变）。
6. 画板模式：实时绘制、清空、保存为默认展示。
7. 俄罗斯方块：保持当前设置页参数。

## 4. 协议策略

采用两阶段，先快落地、后固件原生优化。

## 4.1 阶段A（先上线）
复用现有命令：
1. `set_mode`
2. `set_clock_config`
3. `animation_begin/frame_begin/frame_status/animation_end`
4. 二进制像素流（5字节像素：`x,y,r,g,b`）

说明：
文本/呼吸/律动先由前端渲染成帧动画，再走现有动画下发链路。

## 4.2 阶段B（固件原生）
新增命令（已确认命名）：
1. `set_text_display`
2. `set_breath_effect`
3. `set_rhythm_effect`
4. `save_canvas` / `load_canvas` / `clear_canvas`

## 5. 参数映射表（新增命令）

以下是“字段 -> 来源”映射，实施前按此确认。

|---|---|

| `speed` | 速度滑杆 |
| `direction` | 方向选择 |

说明：
1. `text`必须支持中文字符输入与显示。
2. 中文显示策略由前端渲染与固件播放协同完成，用户只在 uniapp 操作参数。

### 5.2 呼吸灯

### 5.3 律动

|---|---|

| 字段 | 来源 |

| `cmd` | 固定值：`clear_canvas` |

## 5.5 预设定义（统一口径）

`预设` = 一组可复用的完整参数模板（不是开发配置）。

示例：
1. 文本展示预设：文本内容 + 字体 + 颜色 + 速度 + 方向 + 循环。
2. 呼吸灯预设：颜色 + 最小亮度 + 最大亮度 + 速度/周期 + 波形。
3. 律动预设：BPM + 强度 + 色组 + 方向 + 循环。

用户操作：
1. 点击预设一键填充参数。
2. 再点击`保存并应用`发送到设备。

## 8. 验收标准

1. 用户点击主功能不会立即切设备，只有保存并应用才切换。
2. 文本展示支持中文，包含速度/循环/方向等参数可由 uniapp 下发控制。
3. 呼吸灯与律动全部开关可在 uniapp 控制并成功下发。
4. 文本/呼吸/律动至少各有 1 套可用预设，并可成功下发。
5. 拼豆退出自动恢复上次业务模式，20 次连续进出无异常。
6. 画板模式从控制页可直接进入，且不再混在时钟编辑器内部。
7. 固件原生命令路径可替代动画回放路径，端到端延迟下降明显。
8. 文档更新项（M5）全部完成。

## 9. 风险与已确认决策

1. 风险：新增命令后协议版本不一致。
   处理：在 `status` 增加能力位，前端按能力位决定走原生或动画回退。

2. 风险：律动效果过于依赖实时时钟，帧率抖动。
   处理：固件侧使用毫秒时间基准，不依赖网络。

3. 已确认决策：
   1. 命令名采用：`set_text_display / set_breath_effect / set_rhythm_effect`。
   2. 文本展示必须支持中文，不采用“仅英文数字”过渡方案。
   3. 所有参数开关由 uniapp 提供并发送控制。

## 10. 详细开发拆解（执行版）

## 10.1 uniapp 端（按页面）

1. `pages/control/control.vue`
   1. 功能卡交互：点击即进入对应编辑页（无二级“进入编辑”按钮）。
   2. 功能卡视觉：全部常亮，不再依赖选中态高亮。
   3. 主操作入口：根据最近业务模式显示“编辑并发送”文案。
2. `pages/clock-editor/clock-editor.vue`
   1. 仅保留时钟相关参数（静态/动态时钟）。
   2. 移除绘制相关入口与绘制数据叠加。
   3. 发送链路：点击保存时下发模式与配置。
3. `pages/effect-editor/effect-editor.vue`
   1. 文本展示：中文文本、方向、速度、循环、颜色。
   2. 呼吸灯：速度、亮度区间、周期、波形、颜色、快速预设。
   3. 律动：BPM、强度、方向、模式、颜色组、快速预设。
   4. 发送策略：优先原生命令，失败回退动画链路。
4. `pages/canvas-editor/canvas-editor.vue`
   1. 绘制/擦除/撤销/重做/缩放/拖拽。
   2. 连接设备时实时预览。
   3. 保存并应用时写入设备持久化（`save_canvas`）。
   4. 清空时清设备内容（`clear_canvas`）并同步本地缓存。
5. `pages/tetris-settings/tetris-settings.vue`
   1. 保持现有参数模型不变。
   2. 保持“保存并应用”语义，不在进入页面时切模式。

## 10.2 固件端（按模块）

1. `src/websocket_command_handlers.cpp`
   1. 新增并维护命令分发：`set_breath_effect`、`set_rhythm_effect`、`save_canvas`、`load_canvas`、`clear_canvas`。
   2. 参数校验失败直接返回错误，不做字段兼容映射。

   1. 新增 native effect 状态结构与渲染入口。

4. `src/main.cpp`
   1. 整合模式调度与 native effect 调度。
   2. 保障从编辑页退出后显示不黑屏。
