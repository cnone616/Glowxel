# 设备显示迁移参考

更新日期：2026-04-14

## 文档目的

这份文档只做一件事：

把你选中的 10 个开源仓库，重新整理成 Glowxel 里可执行的功能分组。

重点不是“它们分别是什么项目”，而是：

1. 它们在我们项目里分别替换什么
2. 哪些是主来源，哪些只是借思路
3. 现阶段页面里的功能应该怎么归类
4. 后面继续迁移时不能再回到旧的粗糙实现

## 成品保护边界

以下模式已经定板，后续迁移参考仓库时不再碰它们：

- `桌面宠物`
- `静态时钟`
- `动态时钟`
- `主题模式`
- `画板模式`

后续允许推倒重来的，只放在这些功能上：

- `俄罗斯方块时钟`
- `像素信息屏`
- `像素屏保`
- `矩阵流光`
- `音频频谱`
- `GIF / 场景播放器底座`

最新定板补充：

- `天气看板 / 倒数看板 / 计时看板 / 提醒看板` 已确认不适合当前 `64×64` 屏幕观感，且页面“不好看”
- 本轮不是继续修样式，而是直接删除对应 uniapp 页面文件，并移除入口与路由
- 后续迁移参考不再把这四页作为当前可用页面保留

执行原则：

- 不再围绕旧的粗糙效果缝缝补补
- 这些待重做功能直接按参考仓库的成熟效果重构
- 若旧实现和参考仓库效果冲突，优先删旧实现，不保留“历史包袱玩法”

## 游戏屏保收口状态

这一组已经确认不再按“演示集”思路推进，而是按正式产品功能收口。

正式保留项：

- `方块时钟`
- `俄罗斯方块`
- `迷宫漫游`
- `贪吃蛇`

降级或移出项：

- `像素乒乓`
  - 结论：降级，不再作为正式产品游戏屏保推进
- `排序矩阵`
  - 结论：降级，不再作为正式产品游戏屏保推进
- `弹跳图标`
  - 结论：移出 `游戏屏保`，后续归到 `像素屏保`

当前执行约束：

- `游戏屏保` 的正式完成标准必须是“板载程序自己跑逻辑”
- `uniapp` 只负责预览、参数编辑、命令下发
- `方块时钟` 继续走现有原生命令链路
- `迷宫漫游 / 贪吃蛇 / 俄罗斯方块` 统一走 `set_game_screensaver`
- `贪吃蛇` 参数语义固定为 `snakeWidth`，表示蛇身宽度，最小值必须支持 `1`
- `迷宫漫游` 参数语义固定为 `mazeSizeMode`，只允许 `wide / dense`
- `Tetris`
- `WatermelonPlasma`
- `Wave`
- `Rain`
- `Sparks`
