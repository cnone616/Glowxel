# 设备显示迁移参考

更新日期：2026-04-07

## 文档目的

这份文档只做一件事：

把你选中的 10 个开源仓库，重新整理成 Glowxel 里可执行的功能分组。

重点不是“它们分别是什么项目”，而是：

1. 它们在我们项目里分别替换什么
2. 哪些是主来源，哪些只是借思路
3. 现阶段页面里的功能应该怎么归类
4. 后面继续迁移时不能再回到旧的粗糙实现

## 总划分

当前我按 5 个大类来划分：

| 大类 | 我们项目里的功能 | 主来源 | 辅来源 |
| --- | --- | --- | --- |
| 1. 俄罗斯方块时钟类 | `俄罗斯方块时钟` | `toblum/esp_p10_tetris_clock` | `sshcrack/led-matrix` |
| 2. 信息屏 / 看板类 | `像素信息屏` `天气看板` `倒数看板` `计时看板` `提醒看板` | `Blueforcer/awtrix3` | `mzashh/HUB75-Pixel-Art-Display` `sshcrack/led-matrix` |
| 3. 屏保 / 场景类 | `像素屏保` `矩阵流光` | `sshcrack/led-matrix` | `wled/WLED` `toggledbits/MatrixFireFast` `poemusica/rpi-matrix-gif` `A223D/oledAnimationsESP32Arduino` |
| 4. 音频频谱类 | `音频频谱` | `s-marley/ESP32_FFT_VU` | `wled/WLED` |

## 仓库逐个归类

### 1. `toblum/esp_p10_tetris_clock`

归类：

- 俄罗斯方块时钟主来源

适合借：

- 方块下落组成当前时间
- 时间变化后重组
- 方块块型、掉落节奏、拼装顺序

对应 Glowxel：

- `俄罗斯方块时钟`

备注：

- 这类功能不要再归到普通屏保
- 它是一个独立主功能，不是“方块风格动画”

### 2. `Blueforcer/awtrix3`

归类：

- 信息屏 / 看板主来源

适合借：

- 信息卡布局
- 小图标动画
- 天气卡、提醒卡、状态卡
- 小屏阅读层级

对应 Glowxel：

- `像素信息屏`
- `天气看板`
- `提醒看板`
- `倒数看板`
- `计时看板`

备注：

- 这一类应该优先做成“可读、清晰、卡片化”
- 不是继续做一堆抽象特效

### 3. `mzashh/HUB75-Pixel-Art-Display`

归类：

- 内容承载底座主来源

适合借：

- GIF / 图片 / 滚动字承载方式
- 时钟和滚动文字的底层组织
- 简单内容展示底座

对应 Glowxel：

- `像素信息屏` 的底层承载方式
- 后续 GIF / 场景播放功能
- 小屏内容轮播底座

备注：

- 它更像“底座项目”
- 不单独对应一个炫技功能

### 4. `sshcrack/led-matrix`

归类：

- 场景类功能主来源

适合借：

- 时钟场景
- 天气场景
- Tetris 场景
- 雨幕、星空、像素场景屏保
- 多种可切换主题化内容

对应 Glowxel：

- `像素屏保`
- `矩阵流光`
- `天气看板`
- `俄罗斯方块时钟`

备注：

- 这是当前最值得反复对标的仓库之一
- 因为它不是单点特效，而是一整套矩阵内容系统

### 5. `wled/WLED`

归类：

- 算法特效辅来源

适合借：

- 流动
- 扫描
- 噪波
- 追光
- 网格扫描

对应 Glowxel：

- `矩阵流光`
- `像素屏保`
- `音频频谱` 的部分节拍表现

备注：

- WLED 不适合直接定义我们的产品功能名
- 但非常适合拿来替掉粗糙算法效果

### 6. `s-marley/ESP32_FFT_VU`

归类：

- 音频频谱主来源

适合借：

- 柱阵频谱
- 峰值保持
- 中心节拍
- VU 表现

对应 Glowxel：

- `音频频谱`

备注：

- 这块不应该再做成泛化动态光效
- 就应该明确叫频谱 / VU / 节拍板

### 7. `toggledbits/MatrixFireFast`

归类：

- 火焰屏保辅来源

适合借：

- 火焰底层算法
- 暖色能量屏保
- 热流型全屏背景

对应 Glowxel：

- `像素屏保`
- `矩阵流光` 里的暖色整屏分支

备注：

- 它不是独立页面功能
- 更像我们场景库里的一个重要子类型

### 8. `poemusica/rpi-matrix-gif`

归类：

- GIF 播放辅来源

适合借：

- GIF 帧播放
- 像素场景屏保素材循环
- 小型动画播放器思路

对应 Glowxel：

- 后续 GIF 播放
- 场景素材库
- 像素动画承载层

备注：

- 这块更偏播放器，不是产品主视觉来源

### 9. `A223D/oledAnimationsESP32Arduino`

归类：

- 小体量动画素材辅来源

适合借：

- alarm
- confetti
- monster
- 小型趣味动画

对应 Glowxel：

- `提醒看板`
- `像素屏保` 的小动画素材

备注：

- 虽然是 OLED，但动画语义能借
- 更适合做小素材，不适合当主架构

### 10. `pixelmatix/SmartMatrix`

归类：

- 分层渲染 / 底层生态辅来源

适合借：

- 分层绘制
- 字体
- 颜色和渲染组织
- HUB75 生态的成熟底层思路

对应 Glowxel：

- 像素内容叠加
- 时钟 / 信息 / 动画混合渲染
- 后续素材和图层系统

备注：

- 它不直接决定页面功能名
- 但会影响底层组织方式

## 当前迁移原则

1. 不再继续维护原先那批粗糙旧效果的命名和思路。
2. 每个功能必须有明确主来源，不能混成“东拼西凑的旧逻辑”。
3. 同一个来源可以支撑多个功能，但每个功能必须说明自己主要对标谁。
4. 能替换旧实现就直接替换，不以“在旧基础上优化”为主路径。
5. 页面名、预设名、预览效果、板载逻辑，最终都要和来源语义对得上。

4. `像素屏保`
