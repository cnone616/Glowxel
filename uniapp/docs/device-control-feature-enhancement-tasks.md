## 4. 本轮明确要做的增强方向

依据参考图与用户确认，本轮设备控制增强目标如下。

### 4.1 保留并增强的功能
1. `静态时钟`
   说明：保持为独立功能，不和动态时钟合并，不重做协议，不随便改现有可用链路。
2. `动态时钟`
   说明：保持为独立功能，不和静态时钟合并，不动你板载程序现有区分方式。
3. `自定义文字`
   说明：基于现有 `text_display` 继续增强，不重做另一套。
4. `呼吸灯`
   说明：保持独立功能，不并入律动。
5. `律动`
   说明：保持独立功能，不并入呼吸灯。
6. `像素画板`
   说明：继续使用当前独立 `canvas-editor`，只做设备控制入口和能力增强，不改掉现有可用链路。

### 4.2 新增的功能
1. `天气`
2. `倒计时`
3. `秒表`
4. `通知提醒`

说明：
1. `图片/动画` 不作为新的独立主功能拆出来。
2. 图片与 GIF 继续归属在静态时钟 / 动态时钟原有能力里，不额外改成功能卡片。

### 4.3 明确不做的功能
1. `番茄时钟`
2. `贪吃蛇`
3. `屏保`

说明：
1. 这里的“不做”指本轮不进入新增计划。
2. `方块屏保` 属于 Glowxel 已有功能，不删除；但不按参考图继续扩展新的“屏保功能族”。

## 5. 产品结构调整原则

### 5.1 绝不破坏原有功能
1. 不删除现有可用页面
2. 不先砍旧功能再补新功能
3. 现有可用命令链路优先保留
4. 现有本地存储 key 不随意改名
5. 静态时钟 / 动态时钟不做合并改造
6. 图片 / GIF 仍留在静态时钟 / 动态时钟内部能力中
7. 呼吸灯 / 律动不做产品层合并

### 5.2 只做“增强式演进”
1. 先补功能分组和入口结构
2. 再补各功能独立编辑页或子面板
3. 旧功能完成迁移前，原入口保留可回退

### 5.3 预览统一规则
1. 设备控制相关页面统一提供 64x64 预览
2. 除画板模式外，预览只负责展示结果，不提供拖拽、缩放、平移交互
3. 画板模式继续保留绘制所需的交互能力
