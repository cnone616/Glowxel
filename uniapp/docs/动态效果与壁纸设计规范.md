# Glowxel 动态效果与壁纸设计规范

更新日期：2026-04-26

说明：本文里的 `Glowxel` 指仓库 / 工程代号；用户可见产品名统一为 `光格像素工坊`，设备名统一为 `Glowxel PixelBoard`。

## 1. 文档定位

本文用于沉淀 `Glowxel` 后续所有动态效果、壁纸、屏保、氛围场景的长期设计与实现规范。

2026-04-26 用户最新纠偏：

- 本文不能只写成“实现边界 / 技术门禁 / 不要做什么”的清单
- 本文还必须承担“参考效果借鉴词典”的职责，方便后续直接拿来做主题开发、样式开发、动态效果开发
- 后续阅读本文时，目标不是只得出“可以分层”，而是要能提炼出“多层、多角度、多维度、多时间”的组合关键词和设计语言
- 后续基于本文做主题方案时，默认要优先输出“结构关键词 + 空间关键词 + 时间关键词 + 材质关键词 + 取舍建议”，而不是只给单层直白实现
- 后续借鉴外部动态效果时，必须先回到源码核对，再经过至少两路视角交叉归纳后沉淀成关键词；禁止只靠单人主观观感总结“像什么”

它解决的不是“某一个效果怎么抄出来”，而是以下四类长期问题：

1. 低分辨率 `64×64` 屏幕下，什么样的动态语言仍然成立
2. 如何把“看起来生动”的效果拆成可复用的图层、运动与色阶方法
3. 哪些能力适合留在 `uniapp` 预览，哪些必须收口成板载可承受方案
4. 后续再接星空、海浪、水世界、霓虹隧道、云层、粒子、宇宙场景时，如何避免重新走回“边做边猜、最后重写”的低效路径

本文是专项规范，不替代以下文档：

- `docs/development-guardrails.md`
  负责项目总边界、设备状态语义、板载边界与专项硬约束
- `docs/neubrutalism-style-system.md`
  负责页面壳层、卡片、控件、信息密度、视觉风格
- `docs/非外接依赖全量参数映射表.md`
  负责协议字段与参数合同

后续凡是“动态效果 / 壁纸 / 屏保 / 氛围场景”的视觉语言、动画语义、低分辨率实现、板载取舍规则，都应优先沉淀到本文。

## 2. 证据来源

本规范基于以下已核对实现整理，不允许脱离源码臆测：

- 当前仓库总边界：`docs/development-guardrails.md`
- 上游视觉参考：`Deep-Fold/PixelPlanets`

本轮设计方法抽象的第一参考，是 `PixelPlanets` 这条线已经验证过的以下事实：

- 生动感主要来自“语义化分层 + 异步运动”，不是来自“整团图案一起转”
- `64×64` 成立的关键是“像素化优先、离散色阶、局部 dither、背景克制”
- 板载长期运行要优先保可读性与稳定性，而不是保留完整生成器自由度

## 3. 总目标

后续所有动态效果与壁纸，默认同时满足以下四个目标：

1. 一眼能看出主体，不是满屏无重点噪声
2. 在 `64×64` 下仍保留结构辨识，不靠想象补细节
3. 运动语义与对象本身匹配，不靠统一假动作糊弄
4. 能走向板载长期运行，不长期停留在高成本前端演示

如果某个方案同时违背以上四条中的两条及以上，应直接降级、收缩或放弃，不继续投入。

## 4. 术语、优先级与预设谱系

### 4.1 信息源优先级

后续分析和实现动态效果时，信息源优先级统一如下：

1. 用户已确认并已写入仓库文档的硬约束
2. 上游参考仓库中真实存在的视觉语义、图层关系和时间语义
3. 当前仓库已经完成的本地预览与板载实现
4. 只有在以上三层都没有结论时，才允许提出待确认项

执行时默认遵守以下判断顺序：

- 如果 `development-guardrails.md` 已有硬约束，优先服从硬约束
- 如果硬约束要求“回到上游源码核对”，则不得拿主观观感替代
- 如果本地实现与上游语义不一致，应优先修本地实现，而不是重定义上游语义
- 如果板载取舍与上游完整实现冲突，应保留语义骨架，裁掉复杂度，不得把对象语义直接改坏

### 4.2 术语定义

为避免后续讨论时混用，本文件统一采用以下术语：

- `主体层`
  画面中承载对象身份的核心层，例如星球主体、海面主波场、主云团、主旋臂
- `覆盖层`
  叠加在主体之上的表面层，例如云层、冰层、裂隙、薄雾、次级水纹
- `强调层`
  用于建立空间感、动态亮点或特殊材质的层，例如星环、吸积盘、耀斑、白浪、焦散亮带
- `背景层`
  主体之外的环境层，默认必须克制，不能与主体争抢焦点
- `内部分层`
  不拆成多个节点，但在一个 shader 或一个算法里，通过分档、分区、分高度、分带状逻辑形成的“伪多层”
- `结构随机`
  改变对象地形、噪声布局、分区位置、轮廓身份的随机
- `配色随机`
  不改对象身份，只改变色盘、色相、亮暗层级的随机
- `运动语义`
  某一层到底是整体旋转、纹理流动、局部涌动、外环环流、径向喷发，还是保持稳定

### 4.3 术语消歧

以下概念后续严禁混用：

- `持续运动` 不等于 `整体轮廓旋转`
- `方向控制` 不等于 `构图角度`
- `结构随机` 不等于 `配色随机`
- `局部层旋转` 不等于 `整颗对象旋转`
- `内部分层` 不等于 `必须拆成多个节点`
- `板载可承受取舍` 不等于 `可以随意改掉视觉语义`

后续讨论中，如果用户说“它要一直在动”，默认优先理解为“运动语义应持续成立”，而不是“整颗对象必须绕中心匀速旋转”。

## 5. 硬约束

### 5.1 设计边界

- 不把 `PixelPlanets` 一类外部仓库当“完整自动生成器”照搬进项目
- 不把运行时通用编辑器、多图层自由组合、大量 shader 参数面板带进正式产品链路
- 板载负担优先级高于参数丰富度
- 优先选择单主体、固定构图、少参数、可复现的方案
- 预览可以丰富，但正式板载实现必须收口

### 5.2 运动边界

- 默认禁止把所有对象都做成“整颗对象统一自转”
- 方向控制优先理解为“时间流向翻转”，不是“外轮廓必须持续旋转”
- 低速档必须靠减小相位增量或流速实现，禁止靠降帧和大步跳动假装变慢
- 如果上游语义是“局部流动”，不得擅自改成“整体在转”
- 如果上游语义是“核心稳定、外层运动”，不得把核心也一起转起来

### 5.3 背景边界

- 太空类背景默认纯黑底
- 仅允许少量 `1` 像素远处星点
- 禁止铺整屏渐层、发光云团、多像素星芒、泛滥光雾
- 背景只能辅助空间感，不能替代主体本身成立

### 5.4 协议与板载边界

- 预览参数不自动等于正式协议参数
- 新效果若进入发送态，必须先给出参数映射表
- 未进入协议的内部渲染参数，不得伪装成正式设备能力
- 全屏动画默认按“离屏绘制 -> 一次提交”设计，禁止边清屏边直绘真实显示缓冲

## 6. 参考拆解与视觉语言方法论

### 6.1 本文的借鉴输出目标

后续当需要借鉴某个外部动态效果时，默认不是只回答“它分了几层”，而是要至少提炼出下面五类可复用输出：

1. `结构关键词`
   这个效果是“主体 + 覆盖”“静核 + 外环”“前景遮挡 + 中景主体 + 远景粒子”，还是“单 shader 伪分层”
2. `空间关键词`
   它依赖球面、极坐标、倾斜环面、扭转平面、偏心光源，还是前后遮挡关系
3. `时间关键词`
   它是同相、错相、快外层慢核心、双时间叠加、极慢底纹，还是整体刚体旋转例外
4. `材质关键词`
   它是云压地表、河道爬行、裂隙发亮、吸积卷吸、旋臂扭转、耀斑喷流，还是白浪翻卷
5. `取舍关键词`
   哪些是必须保留的语义骨架，哪些是可以固定、裁掉、简化的复杂度

如果只提炼出“它有很多层”，但说不清上面五类关键词，说明拆解还不够可复用。

### 6.2 参考拆解五层输出法

后续借鉴任何动态效果，默认按下面五层往下拆：

| 层级 | 要回答的问题 | 输出示例 |
| --- | --- | --- |
| `主题层` | 它想传达什么气质 | 冷寂、卷吸、潮湿、燃烧、悬浮、漂移 |
| `结构层` | 画面由哪些角色组成 | 主体、覆盖、强调、遮挡、远景 |
| `空间层` | 这些角色在什么坐标空间中成立 | 球面、极坐标、倾斜环面、扭转平面 |
| `时间层` | 各角色如何运动、谁快谁慢 | 慢主体快细节、双时间叠加、反向流 |
| `实现层` | 真正的低分辨率手段是什么 | 量化、色阶、dither、排斥半径、缓存 |

这五层输出完之后，才能进入“怎么做成 Glowxel 版本”的阶段。

### 6.3 层叠关键词库

后续做主题开发时，优先从下面的层角色词里挑组合，而不是从抽象风格词直接开做：

| 关键词 | 含义 | 常见作用 |
| --- | --- | --- |
| `基底层` | 提供主体承载底色或底纹 | 水底、气体底带、深空底色 |
| `主体层` | 决定对象身份的主要轮廓或形体 | 星球、云团、海面主波场 |
| `覆盖层` | 压在主体上的表面层 | 云层、冰层、裂隙、薄雾 |
| `强调层` | 建立视觉焦点与动态亮点 | 星环、耀斑、白浪、焦散 |
| `遮挡层` | 制造前后关系或穿插感 | 行星前后穿插的环、雾幕、前景剪影 |
| `外晕层` | 扩大主体气场，不直接承担主体识别 | 恒星外 blob、能量晕圈 |
| `粒子层` | 提供稀疏环境生命感 | 星点、漂浮尘、海中悬浮颗粒 |
| `远景层` | 只负责空间深度，不抢主体 | 远处星点、远处云影 |
| `静核层` | 必须保持稳定的核心 | 黑洞核心、能量核、主灯核 |
| `极慢层` | 允许变化，但变化慢到不破坏身份 | 陨坑底纹、远景暗流 |

默认不建议只做单层主体。需要明显生命感的主题，至少先考虑：

- `主体层 + 覆盖层 + 远景层`
- `主体层 + 强调层 + 背景层`
- `静核层 + 外动强调层 + 稀疏粒子层`

### 6.4 空间与角度关键词库

后续做主题开发时，除了“画什么”，还要先决定“在哪种空间里画”：

| 关键词 | 含义 | 适合对象 |
| --- | --- | --- |
| `球面包裹` | 纹理被包到球体表面 | 星球、球形能量体 |
| `倾斜环面` | 围绕主体的扁环平面 | 星环、吸积盘、轨道盘 |
| `极坐标喷发` | 以中心向外或向内组织纹理 | 耀斑、漩涡、卷吸、喷流 |
| `扭转平面` | 平面内容围绕中心扭转 | 星系、旋臂、风暴云团 |
| `偏心光源` | 光照中心不在几何中心 | 更强体积感与方向感 |
| `前后遮挡` | 明确处理谁在前谁在后 | 环从主体后方穿过、雾从前景掠过 |
| `排斥半径` | 背景或粒子不能贴近主体轮廓 | 主体边缘更干净 |
| `外扩比例` | 某层比主体大一圈或数圈 | 星环、外晕、耀斑、波纹扩散 |

如果后续效果显得“平”，优先检查是不是只用了普通平面 UV，没建立正确的空间关键词。

### 6.5 时间与节奏关键词库

后续做主题开发时，默认不要只写“速度快 / 慢”，而是从下面这些时间关系词里选：

| 关键词 | 含义 | 适用例子 |
| --- | --- | --- |
| `同相` | 多层同时同向同速 | 只适合少量统一材质，不适合复杂生命感 |
| `错相` | 多层共享方向，但相位不同 | 云层与表面、双气带 |
| `慢主体快细节` | 大轮廓稳，小纹理活 | 云海、海面、行星表面 |
| `快外层慢核心` | 外围更活跃，中心更稳定 | 黑洞、恒星、能量核 |
| `双时间叠加` | 同一层内部有两套时间 | wobble + flow、主波 + 细波 |
| `反向流` | 不同层方向相反 | 内外旋臂、表层与底层逆流 |
| `极慢底纹` | 某层几乎静止，但不绝对死 | 陨坑、远景暗纹、深层流 |
| `连续低速` | 低速仍然连续，不靠降帧 | 所有正式低速档 |
| `闭环回放` | 首尾循环平滑闭合 | 自动播放类预览、屏保类效果 |
| `刚体例外` | 整体轮廓旋转本身就是语义 | 小行星、碎片、少数机械体 |

需要明显生命感时，默认至少组合其中两条，不要只给一个匀速线性时间。

### 6.6 材质与气氛关键词库

后续做主题或样式开发时，建议直接从材质动作词入手，而不是只写“做个宇宙风 / 海洋风”：

| 关键词 | 语义 | 可迁移方向 |
| --- | --- | --- |
| `云压地表` | 覆盖层压在主体上，形成天气感 | 星球、山海、云海 |
| `河道爬行` | 亮暗细线沿表面缓慢游走 | 类地、海底纹理、电子纹 |
| `裂隙发亮` | 深色主体里有亮线破开 | 熔岩、赛博裂纹、能量岩层 |
| `环材流动` | 环几何稳定，材质在环内滑动 | 星环、轨道、能量环 |
| `吸积卷吸` | 材质向中心卷入或沿盘流动 | 黑洞、漩涡、风眼 |
| `旋臂扭转` | 分层带状结构围中心扭动 | 星系、云涡、风暴带 |
| `耀斑喷流` | 从核心向外放射性的短促强运动 | 恒星、火焰、能量喷口 |
| `稀疏漂粒` | 少量粒子补充环境生命感 | 深空、海水、雾夜 |
| `冷暖芯边对比` | 核心和边缘的温差/色差 | 恒星、能量核、霓虹主题 |
| `明暗分区压缩` | 不是渐变，而是块状层次切换 | 低分辨率宇宙体、云团、波场 |

### 6.7 多维叠加检查

后续如果希望主题不显“扁、假、散”，默认至少从以下六个维度里启用三到四个：

| 维度 | 问题 |
| --- | --- |
| `层维度` | 是否至少有主体之外的第二角色层 |
| `空间维度` | 是否选对了坐标空间，而不是只用平面 UV |
| `时间维度` | 是否至少有一组错相或速度对比 |
| `材质维度` | 是否有明确材质动作词，而不是单纯噪声滚动 |
| `光照维度` | 是否有偏心光、前后遮挡、深浅分区 |
| `尺度维度` | 是否有大结构、中结构、小细节的层次差 |

只有一个维度在工作时，效果通常会显得单薄；三到四个维度同时成立时，主题才更容易有生命感。

### 6.8 主题开发组合语法

后续做新主题时，可以直接从下面这些“组合句法”出发：

- `稳定主体 + 覆盖漂移 + 稀疏远景`
- `静核 + 快环 + 极慢底纹`
- `主体深色 + 裂隙亮线 + 漂浮颗粒`
- `前景遮挡 + 中景主体 + 后景粒子`
- `大轮廓慢 + 中结构中 + 小细节快`
- `偏心光源 + 深浅分区 + 局部 dither`
- `外晕扩张 + 核心稳定 + 径向喷流`

后续如果我基于本文给你做某个主题方案，默认应该至少给出两到三组这样的组合语法，而不是只给一句“做成多层一些”。

### 6.9 先定主体，再做细节

后续新效果必须先回答这三个问题：

1. 主体是什么
2. 主体边界在哪里
3. 用户第一眼该先看见什么

如果一个效果必须靠文案解释才能看懂主体，视为主体设计失败。

### 6.10 分层必须有语义

严禁“为了复杂而多层”。每一层都必须能回答：

1. 它代表什么材质或空间角色
2. 它为什么要独立成层
3. 它与主体的前后关系是什么
4. 它的运动是否应该独立

可复用的语义分层模型如下：

| 模型 | 典型结构 | 适用效果 |
| --- | --- | --- |
| `主体 + 覆盖` | 地表 + 云层 | 湿润类地、雾层山体、云海 |
| `底层 + 主体 + 覆盖` | 水面底色 + 岛屿/冰面 + 云层 | 群岛、冰世界、浅海地貌 |
| `主体 + 强调层` | 气态主体 + 星环 | 行星环、能量环、轨道带 |
| `稳定核心 + 运动外层` | 黑洞核心 + 吸积盘 | 黑洞、能量核心、漩涡源 |
| `单 shader 内部分层` | 旋臂 + 倾斜层高 + 深浅档位 | 星系、云团、分层波场 |
| `刚体例外` | 不规则实体整体旋转 | 小行星、碎片、机械构件 |

### 6.11 主体稳定，小结构活着

高质量动态效果应优先满足：

- 主体身份稳定
- 局部细节活跃
- 层间关系清晰
- 首尾循环平滑

应避免：

- 每几秒像换了一个对象
- 结构在随机漂移，导致同一种子失去身份感
- 局部层乱动，破坏主体可读性
- 首尾跳变，让方向像反了或像中途回弹

### 6.12 主题核句型库

后续基于本文立项新主题时，开头不再写空泛风格词，默认先用下面这些句型定题：

- `本主题以【主体对象】为核心，传达【气质词 1 / 气质词 2】，第一眼焦点是【主体识别点】，避免【反主题倾向】。`
- `本主题的空间身份依赖【主空间】，动态生命感来自【主时间句法】，不依赖【错误替代动作】。`
- `本主题的板载语义骨架是【必须保留的层关系】，即使收口也不能退化成【禁止退化项】。`

后续输出主题方案时，`主题核` 最少必须写清以下字段：

| 字段 | 含义 |
| --- | --- |
| `主体对象` | 画面里真正承载身份的对象 |
| `气质词` | 冷寂、卷吸、潮湿、漂移、燃烧、悬浮等 |
| `第一眼焦点` | 用户第一眼应该先看见什么 |
| `反主题倾向` | 本主题最怕做成什么 |
| `参考原型` | 应先归到哪类原型模式 |

### 6.13 镜头与构图模板库

“角度”后续不再只写成坐标类型，还要落成可直接复用的镜头模板：

| 模板 | 结构句法 | 空间句法 | 主要作用 | 适合题材 |
| --- | --- | --- | --- | --- |
| `正视居中核` | `静核层 + 外动层 + 稀疏远景` | 主体居中，外层围核展开，偏心光只做体积感 | 建立稳定中心和明确身份 | 黑洞、恒星、能量核、灯塔核 |
| `偏心主体留黑` | `主体层 + 副流层 + 远景层` | 主体偏离中心，一侧留黑，另一侧保副流或余辉 | 建立方向感、压迫感、呼吸空间 | 风暴眼、海面月球、霓虹核心 |
| `前景掠过遮挡` | `前景遮挡层 + 中景主体 + 后景粒子` | 前景贴近屏幕快速掠过，中景承载主体，后景只保稀疏点 | 建立纵深和“贴屏感” | 海底、森林月面、雾夜、窗外景 |
| `斜切环绕` | `主体层 + 环面强调层 + 遮挡关系` | 倾斜环面从主体后侧穿入、前侧穿出 | 建立伪三维和前后层次 | 星环、轨道盘、传送门、机械环 |
| `扭转穿行` | `扭转平面 + 前景掠线 + 中心亮区` | 大平面围中心扭转，局部前景层从边缘切过 | 建立通道感、卷吸感、长纵深 | 星系、云旋臂、霓虹隧道、水流隧道 |

后续只要主题显得“像一个居中图标”，默认优先换镜头模板，而不是先加更多细节。

### 6.14 时间节奏句法库

下面这些倍率是相对节奏配方，不是协议字段；用途是帮助后续快速写出“谁快谁慢谁稳”的时间语言：

| 句法 | 相对节奏配方 | 语义重点 | 适合主题 |
| --- | --- | --- | --- |
| `同向错相三层` | `主体 1.00x / 覆盖 0.82x / 细节 1.15x` | 方向一致，但相位和速度不重合 | 云层、双气带、三层地貌 |
| `慢主体 + 快覆盖 + 极慢底纹` | `主体 0.55x / 覆盖 1.40x / 底纹 0.15x` | 身份稳定，表面活着，深层几乎不扰动主体 | 海面、云海、沙丘、风暴带 |
| `静核 + 快环 + 副脉冲` | `核心 0x / 环层 1.50x / 脉冲 2.20x` | 核心必须稳，外围靠快层和短脉冲建立活性 | 黑洞、传送门、能量核 |
| `双时间叠加层` | `主流 0.85x + 摆动 0.12x` | 同一层内部同时拥有主流向和低频 wobble | 吸积盘、海流、气带、能量环 |
| `反向流双层` | `内层 0.90x / 外层 -1.10x` | 两层方向相反，形成张力和卷吸感 | 双环、双旋臂、风暴眼 |
| `连续低速层` | `0.10x - 0.30x` | 极慢但连续，禁止靠降帧伪装慢速 | 远景暗流、陨坑、深海底纹 |

如果一个方案写不出“哪层 0x、哪层 0.5x、哪层 1.5x”，说明时间语义还不够清楚。

### 6.15 光照与明暗句法库

后续做主题开发时，光照不再只理解成“变亮一点”，而是作为独立维度参与主题气质设计：

| 句法 | 核心做法 | 主要气质 | 低分辨率提醒 |
| --- | --- | --- | --- |
| `偏心侧光` | 高光中心故意偏离几何中心，让一侧更亮、一侧更沉 | 体积感、方向感、压迫感 | 高光不做软大渐变，优先切成明确亮区 |
| `背光轮廓` | 主体内部偏暗，边缘亮轮廓抬起轮廓识别 | 冷寂、孤立、漂浮 | 边缘亮线要薄，避免整圈发光糊边 |
| `冷核暖边` | 核心偏冷或偏白，边缘用更暖或更亮的活动层包围 | 能量感、核心感、超现实 | 冷暖差优先靠色阶对比，不靠柔光混色 |
| `中心暗边缘亮` | 主体中心压暗，外缘或环带承担主要亮度 | 卷吸、空洞、悬浮 | 中心暗区必须保持可读，不可直接黑成洞 |
| `亮线破面` | 在深色主体中用少量亮线切开大面 | 裂隙、岩浆、电子纹、机械能量缝 | 亮线数量要克制，避免满屏碎亮点 |
| `明暗分区压缩` | 用块状明暗区替代长渐变，压出结构层次 | 像素感、厚重感、低分辨率友好 | 分区数优先少而准，不追求平滑过渡 |

### 6.16 尺度分工模板

后续凡是需要“更活”的主题，默认至少拆成大、中、小三档尺度，不要所有细节同一级别抢戏：

| 尺度档 | 默认职责 | 默认速度 | 常见载体 | 失效表现 |
| --- | --- | --- | --- | --- |
| `大轮廓` | 承担第一眼识别、主体边界、主要块面 | `0x - 0.60x` | 星球轮廓、主波面、主云团、主体暗面 | 大轮廓乱动，身份感立刻变差 |
| `中结构` | 承担主要运动语义和层次转折 | `0.60x - 1.20x` | 气带、海浪主纹、旋臂、云带、环材 | 中结构没有节奏对比，画面会发平 |
| `小细节` | 承担活性、闪点、粒子、边缘破碎感 | `1.20x - 2.50x` | 高光点、耀斑、浪尖、漂粒、裂隙尖端 | 小细节过多会把主体打碎 |

后续优先复用以下三条尺度句法：

- `大轮廓慢 + 中结构中 + 小高光快`
- `大平面稳 + 中条带流 + 小粒子漂`
- `大暗面稳 + 中亮带动 + 小爆闪点`

### 6.17 非球体迁移语法

后续从 `PixelPlanets` 借鉴时，不是把“星球”换成别的外形，而是迁移它已经成立的结构句法、空间句法和时间句法：

| 题材方向 | 可迁移的结构语法 | 可迁移的空间语法 | 可迁移的时间语法 | 推荐材质词 |
| --- | --- | --- | --- | --- |
| `海浪 / 海面` | `基底层 + 主波层 + 白浪强调层 + 漂粒层` | `扭转平面 + 平面纵深 + 前景掠过` | `慢主体 + 快浪尖 + 极慢底纹` | `主波推进` `白浪翻卷` `焦散闪烁` |
| `云海 / 沙丘` | `主体层 + 覆盖层 + 远景暗流` | `倾斜平面 + 偏心侧光 + 留黑` | `同向错相三层` | `云压地表` `边缘积光` `稀疏漂粒` |
| `霓虹隧道 / 能量通道` | `静核层 + 双环层 + 前景掠线` | `极坐标 + 倾斜环面 + 扭转穿行` | `静核 + 反向流双层 + 副脉冲` | `环材流动` `径向喷流` `边缘霓虹` |
| `海底 / 水世界` | `前景遮挡层 + 中景主体 + 后景粒子` | `前后遮挡 + 偏心光源 + 排斥半径` | `慢主体 + 快漂粒 + 极慢底纹` | `稀疏漂粒` `冷暖芯边对比` `暗流游动` |
| `风暴眼 / 漩涡` | `静核层 + 主旋带 + 副旋带` | `扭转平面 + 极坐标卷吸 + 偏心主体` | `双时间叠加 + 反向流` | `卷吸` `旋臂扭转` `局部爆闪` |

后续只要是非球体主题，默认先从上表选一条迁移语法，再补题材素材，不要从“题材名字”直接反推实现。

## 7. 动画语义规范

### 7.1 不同层必须拥有不同时间语义

“生动”最可靠的来源，不是更多噪声，而是更清晰的时间分工。

后续设计时，每一层都要先定义：

| 字段 | 含义 |
| --- | --- |
| `motionType` | 整体旋转 / 纹理平移 / 局部环流 / 径向喷发 / 静止 |
| `timeFormula` | 时间如何进入算法 |
| `speedSource` | 速度来自固定 preset、用户步进器，还是内部系数 |
| `directionSource` | 方向是否由全局正负翻转控制 |
| `loopRule` | 首尾是否要求平滑闭合 |

### 7.2 默认动作类型

后续没有充分证据前，优先采用以下默认判断：

- 地表、水面、云层、河道、冰层、熔岩纹理：优先做纹理流动，不做整体刚体旋转
- 星环、吸积盘、能量环：优先做环内材质流动，几何形状只做固定透视
- 黑洞、能量核心、发光核心：优先核心稳定、外层运动
- 星系、云旋臂、风暴带：优先内部扭转、分层漂移，不拆成硬刚体
- 小行星、碎片、孤立机械体：可以考虑整体旋转，但必须先证明这样更符合对象语义

### 7.3 可复用时间模型

后续动态效果优先从以下时间模型中选，不要每次重新发明：

- `线性纹理流动`
  适合云层、水流、地表条带、风暴带
- `环形材质流动`
  适合星环、吸积盘、轨道带、旋转能量环
- `径向外喷 / 向内卷吸`
  适合耀斑、黑洞吸积、爆裂条纹、能量喷涌
- `扭转 + 分层`
  适合星系、漩涡云团、风场、波带
- `极慢层`
  适合陨坑阴影、深层底纹、远景层，不必完全静止，但必须足够稳

### 7.4 低速运动规范

低速档最容易做坏。后续必须遵守：

- 低速仍需连续
- 低速只减小单次位移或相位增量
- 禁止减少刷新频率制造“慢”的错觉
- 禁止每次更新都跨大步，造成明显顿挫

## 8. 坐标空间规范

动态效果不能默认全都在普通二维平面上做。

后续设计时，必须先明确对象使用哪种坐标逻辑：

| 坐标空间 | 典型用途 | 设计提醒 |
| --- | --- | --- |
| `平面 UV` | 雨幕、基础噪声、普通背景流动 | 最简单，但最容易显平 |
| `球面投影` | 星球、球形能量体、天体表面 | 先像素化，再做球面映射 |
| `极坐标` | 耀斑、旋涡、喷流、放射条纹 | 适合做向心/离心运动 |
| `倾斜环面` | 星环、吸积盘、轨道盘 | 几何形状与材质流动要拆开 |
| `倾斜平面 + 扭转` | 星系、云团、旋臂 | 适合做伪三维与分层深度 |

如果一个效果看起来“什么都在转”，常见原因不是时间错，而是坐标空间选错。

## 9. 低分辨率实现规范

### 9.1 像素化顺序

默认顺序应为：

1. 先量化像素坐标
2. 再做球面或其他空间映射
3. 再做噪声、光照、分层与颜色分类

禁止反过来先做连续采样，再在最后硬截成像素，否则结构会先糊，再被放大。

### 9.2 色阶优先于渐变

`64×64` 下，连续渐变常常比离散色阶更脏。

后续优先策略：

- 优先少量明确色阶
- 通过阈值切换建立层次
- 在边界处局部 dither
- 把明暗关系做成结构，而不是做成大面积柔光

### 9.3 dither 使用规则

`dither` 只用于以下场景：

- 色阶交界处的过渡
- 局部材质颗粒感
- 低分辨率下保持边界层次

禁止把 `dither` 当作：

- 整体脏化滤镜
- 掩盖结构不清的补丁
- 假装“更有细节”的万能药

### 9.4 背景密度控制

背景永远服从主体，不允许抢戏。

空间类效果默认遵守：

- 背景星点稀疏
- 背景亮度低于主体
- 主体附近应有排斥半径，避免背景贴脸干扰轮廓
- 不靠背景装饰弥补主体本身无聊

## 10. 随机与可复现规范

### 10.1 随机必须拆成两类

后续新效果默认拆分：

- `seed`
  结构随机，控制对象身份
- `colorSeed`
  配色随机，控制氛围变化

如果后续把这两者混在一起，会直接导致以下问题：

- 一改颜色，结构也变
- 同一对象无法保身份感
- 预览与恢复结果不一致
- 无法稳定复现用户选中的“那一版”

### 10.2 可复现是硬要求

同一套配置下，结果必须一致。

对动态效果来说，这意味着：

- 结构布局一致
- 色盘一致
- 背景分布一致
- 循环方向一致
- 同种子左右方向下仍保持“同一对象”身份感

### 10.3 缓存使用原则

适合缓存的对象：

- 结构不变的背景层
- 同 `preset + seed + colorSeed + size + direction + speed` 下可复用的预览结果
- 仅依赖种子和 preset 的色盘结果

不适合缓存的对象：

- 实时依赖外部输入的交互层
- 明确需要逐帧变化的板载状态层

## 11. 参数设计规范

### 11.1 参数必须先分类

后续新增动态效果参数时，必须先分成三类：

| 参数类别 | 用途 | 例子 |
| --- | --- | --- |
| `可见预览参数` | 用户直接理解并操作 | 类型、大小、方向、速度、随机 |
| `协议正式参数` | 发送到设备后确实生效 | `preset` `size` `direction` `speed` `seed` `colorSeed` |
| `内部固定参数` | 只服务渲染实现，不直接暴露 | `pixels` `dither` `rotationOffset` `octaves` `ringPerspective` |

### 11.2 参数暴露原则

- 优先暴露用户能理解的语义参数
- 不要把 shader 内部实现细节直接暴露成用户参数
- 如果一个参数对用户几乎没有判断价值，应固定在内部
- 如果一个参数只适合预览阶段，不得暗示它已进入正式协议

### 11.3 协议前置要求

只要某效果准备进入发送态，必须先写清：

1. `字段名 -> 来源`
2. 哪些参数进入协议
3. 哪些参数只留预览
4. 板载默认值是什么
5. 前端、WebSocket、固件三端是否一致

## 12. 板载可承受取舍规范

### 12.1 取舍优先级

板载实现的优先级始终是：

1. 主体成立
2. 运动语义正确
3. 循环稳定
4. 不闪、不崩、不拖主循环
5. 细节丰富度

只要 1 到 4 没站住，就不应该为了“多给几个参数”继续加复杂度。

### 12.2 不适合板载直接照搬的内容

- 完整 procedural 编辑器语义
- 高自由度多图层运行时拼装
- 大量实时颜色生成器
- 高密度背景特效
- 通过模式内亮度压制掩盖稳定性问题
- 边清边画真实显示缓冲

### 12.3 允许降级 / 禁止降级矩阵

| 类别 | 允许收口 | 禁止收口 |
| --- | --- | --- |
| 参数收口 | 隐藏预览期低价值参数，固定内部默认值 | 把本已进入正式协议的语义参数静默删掉 |
| 图层收口 | 在不破坏语义前提下减少背景装饰层 | 把多层语义直接压成“整颗对象统一旋转” |
| 配色收口 | 减少色盘自由度，保留层级关系 | 把结构随机和配色随机重新耦合 |
| 背景收口 | 收缩为纯黑底和稀疏 `1px` 星点 | 用大面积氛围层抢主体焦点 |
| 板载优化 | 固定 `pixels`、`dither`、内部噪声细节 | 把设备亮度预算、软启动、模式内亮度压制写进视觉规范 |
| 提交链优化 | 继续离屏绘制后一次提交 | 回退成真实显示缓冲边清边画 |

### 12.4 板载复杂度自检

后续每接一个新效果，至少回答以下问题：

1. 每帧需要几次全屏遍历
2. 每像素大约要做多少次噪声采样
3. 是否有多层独立 pass
4. 是否存在可提前离线或固定化的部分
5. 哪些层可以保留，哪些层应先裁掉

## 13. 原型模式库

后续新效果优先从以下原型出发，不要直接从空白开始：

| 原型 | 说明 | 当前例子 |
| --- | --- | --- |
| `表面流动型` | 主体稳定，表面细节缓慢流动 | `terran_wet` `terran_dry` |
| `三层地貌型` | 底层 + 主体 + 覆盖层 | `islands` `ice_world` |
| `材质裂隙型` | 主体稳，强调层做亮线或裂隙 | `lava_world` |
| `环带强调型` | 主体与环独立运动 | `gas_giant_2` |
| `静核外动型` | 核心稳定，外围强运动 | `black_hole` |
| `内部伪分层型` | 单 shader 内建多层深度 | `galaxy` |
| `刚体例外型` | 整体旋转才合理 | `asteroid` |
| `多坐标复合型` | 外圈团块 + 内核 + 径向耀斑 | `star` |

### 13.1 主题母题模板库

下面这组模板不是“多加一层”的空话，而是后续可以直接复用的主题母题。默认先选母题，再填具体题材：

| 母题 | 结构组合 | 镜头 / 空间 | 时间组合 | 材质关键词 | 适合主题 | 板载收口建议 |
| --- | --- | --- | --- | --- | --- | --- |
| `前景遮挡 + 中景主体 + 后景稀疏粒子` | 前景层只做遮挡，中景承载主体，后景只保稀疏漂粒 | `前景掠过遮挡` + `排斥半径` | `主体 0.6x / 遮挡 1.3x / 后景 0.2x` | `雾幕掠过` `漂粒悬浮` `边缘积光` | 海底、雾夜、森林月面、窗外景 | 前景遮挡层优先做低成本剪影或条带，不做高密度透明雾 |
| `偏心主体 + 一侧留黑 + 反方向副流` | 主体不居中，留黑侧只保极少环境信息 | `偏心主体留黑` | `主体 0.7x / 副流 -1.0x` | `余辉拖带` `暗面压缩` `反向副流` | 风暴眼、月面海景、霓虹核心 | 留黑侧不要再塞装饰层，优先把预算留给主体与副流 |
| `慢主体 + 快覆盖 + 极慢底纹` | 大轮廓稳定，覆盖层承担主要活性，底纹几乎不动 | `平面纵深` 或 `球面包裹` | `主体 0.55x / 覆盖 1.40x / 底纹 0.15x` | `云压地表` `白浪翻卷` `暗流游动` | 海面、云海、沙丘、风暴带 | 极慢层可固定细节规模，避免板载多层高成本噪声 |
| `静核 + 双环层 + 反向流` | 核心稳定，内外两层环承担方向冲突和张力 | `正视居中核` 或 `斜切环绕` | `核心 0x / 内环 0.9x / 外环 -1.1x` | `环材流动` `卷吸` `边缘脉冲` | 黑洞、传送门、机械轨道、赛博天体 | 环几何保持固定，优先只让环内材质流动 |
| `扭转平面 + 前景掠过 + 局部爆闪强调` | 大平面扭转，中景流动，局部强调层短促点亮 | `扭转穿行` | `主平面 0.8x / 前景 1.5x / 爆闪 2.2x` | `旋臂扭转` `局部爆闪` `掠线切过` | 星系、云团、霓虹隧道、水流隧道 | 爆闪必须稀疏，优先局部触发，不做整屏频闪 |
| `大轮廓慢 + 中结构中 + 小高光快` | 先拆尺度，再决定层角色 | 任意主空间，但必须保层级 | `大 0.35x / 中 1.0x / 小 2.0x` | `主面稳` `中带流` `高光跳` | 几乎所有需要生命感的主题 | 小高光要少，先保主体完整度，再给快细节 |

### 13.2 原型迁移速查

如果后续主题不是天体，也不要直接脱离现有原型重想；优先按下表做迁移：

| 当前原型 | 可迁移题材 | 必须保留的语义骨架 | 不要误做成 |
| --- | --- | --- | --- |
| `表面流动型` | 海面、云海、沙丘、电子纹地表 | 主体稳，表面流动 | 整体大平面匀速平移 |
| `三层地貌型` | 海岸、浅海、雪原、台地 | 底层 + 主体 + 覆盖层三层异步 | 三层同速同相 |
| `材质裂隙型` | 熔岩、赛博裂纹、能量岩层 | 深色主体 + 亮线破面 | 满屏泛光贴图 |
| `环带强调型` | 轨道、传送门、机械环、能量圈 | 主体与环分工，前后遮挡成立 | 把整环当刚体整块旋转 |
| `静核外动型` | 能量井、风暴眼、灯塔核 | 核心稳定，外层承担主运动 | 核心也跟着一起转 |
| `内部伪分层型` | 云涡、潮汐涡旋、隧道场景 | 单算法内部建立深度和分层 | 只做一层平面噪声平移 |
| `多坐标复合型` | 恒星风暴、霓虹核心、水流通道 | 外晕、核心、径向强调多坐标并存 | 所有运动都压成同一层 |

## 14. 新效果立项流程

后续任何动态效果 / 壁纸立项，默认按以下流程执行：

1. 先选参考源
2. 先做“来源映射表”，写清楚借谁、借什么、不借什么
3. 先做“图层角色表”
4. 先做“运动语义表”
5. 先做“参数分类表”
6. 先判断板载是否值得接
7. 再开始写预览
8. 预览通过后，再做协议与板载收口

如果跳过前四步直接开写，后续大概率会重新返工。

### 14.1 立项必写五表

后续所有新动态效果，立项时默认先写以下五张表：

| 表名 | 必填内容 | 目的 |
| --- | --- | --- |
| `层角色表` | 主体层、覆盖层、强调层、背景层、遮挡层 | 防止“多层但没语义” |
| `动作归属表` | 哪层在动、哪层必须稳、动作类型、方向来源 | 防止误做整体旋转 |
| `坐标系表` | 平面 UV、球面、极坐标、倾斜平面、环面 | 防止坐标系选错 |
| `低分辨率色阶表` | 色阶数、dither 用法、背景密度、轮廓保护 | 防止 `64×64` 发糊 |
| `板载裁剪表` | 哪些保留、哪些固定、哪些不进协议、哪些先不做 | 防止板载优化侵蚀视觉语义 |

### 14.2 基于本文生成主题方案的默认输出格式

后续如果用户让我“基于这份规范做一个主题 / 壁纸 / 动效方向”，默认应按以下格式输出，而不是只给笼统描述：

1. `主题核`
   一句话说明这次要传达的气质与主体
2. `结构组合`
   主体层、覆盖层、强调层、背景层分别是什么
3. `空间组合`
   使用哪种坐标空间，哪里需要倾角、遮挡、外扩比例
4. `时间组合`
   哪层快、哪层慢、哪层稳、是否有双时间叠加
5. `材质关键词`
   至少三到五个具体材质动作词
6. `低分辨率策略`
   色阶、dither、轮廓保护、背景密度如何处理
7. `板载收口建议`
   哪些必须保留，哪些可固定，哪些不进入协议

如果后续输出里没有第 2 到第 5 项，说明方案仍然过于平面，不足以支持高质量主题开发。

后续凡是基于本文产出新主题，默认至少按下面这张卡片填写；缺其中任意关键字段，视为方案还不够成熟：

| 字段 | 必填内容 |

| `主题核` | `主体对象 + 气质词 + 第一眼焦点 + 反主题倾向` |

## 15. 交付前检查清单

### 15.1 视觉检查

- 主体是否一眼可见
- 背景是否克制
- 是否存在与主体争焦点的装饰层
- 是否在 `64×64` 下仍能辨认层次

### 15.2 动画检查

- 哪些层在动，哪些层不动，是否说得清
- 运动是否符合对象语义
- 是否误把局部流动做成整体旋转
- 首尾循环是否平滑闭合

### 15.3 低分辨率检查

- 像素化是否发生在正确阶段
- 色阶是否足够清晰
- dither 是否只用在边界和必要材质
- 小屏下是否仍有轮廓和明暗分区

### 15.4 板载检查

- 是否需要离屏绘制
- 是否引入过多 pass 或过高采样成本
- 是否存在仅为预览好看而不适合上板的层
- 是否把本应固定的内部参数错误暴露为协议能力

### 15.5 文档检查

- 是否更新来源映射
- 是否更新参数映射
- 是否把新增约束沉淀到规范，而不是留在对话里

## 16. 反模式清单

后续开发中，以下写法视为不合格：

- 把“像星球”理解成“整球一直在转”
- 用更复杂的背景掩盖主体无聊
- 通过不断随机结构伪装生动
- 用降帧造成“慢速”错觉
- 用连续渐变硬压进 `64×64`
- 让用户直接面对大量底层 shader 参数
- 在预览里能调很多，板载却没有对应语义
- 用模式内亮度封顶掩盖真实渲染问题

## 17. 当前星球线专项结论

当前 `星球屏保` 这条线，可以作为后续动态效果的第一批标准样本：

- `islands`
  适合当“表面三层异步”的模板
- `gas_giant_2`
  适合当“主体 + 环带强调层”的模板
- `black_hole`
  适合当“静核外动”的模板
- `star`
  适合当“多坐标复合运动”的模板
- `galaxy`
  适合当“单 shader 内部分层”的模板
- `asteroid`
  适合当“整体刚体旋转例外”的模板

后续若新效果无法判断采用哪一类结构，优先先归到上面某一类，再展开实现，不要直接在空白纸上乱试。

## 18. 后续维护规则

- 新增动态效果设计规范时，优先更新本文

- 若后续继续借鉴外部动态效果，默认流程是 `源码核对 -> 至少两路视角交叉归纳 -> 抽取结构 / 空间 / 时间 / 材质关键词 -> 再进入 Glowxel 主题方案`，不允许只凭单人主观观感直接写方案
