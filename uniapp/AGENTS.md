# Glowxel 仓库入口约束

以下规则为本仓库进入后必须先遵守的硬约束。

## 1. 回复与执行

- 始终使用简体中文回复。
- 用户新增的负面反馈、禁令、修正意见，必须先总结成规则并写入对应文档，再继续改代码。
- 样式、布局、交互表现类要求优先写入 `docs/neubrutalism-style-system.md`。
- 连接、设备状态、传输、模式切换、页面状态语义类要求优先写入 `docs/development-guardrails.md`。

## 2. API 与字段硬约束

- 仅允许使用需求中明确给出的字段名与取值来源，禁止自行发明字段、别名字段或兼容字段。
- 禁止任何兜底写法，包括 `a || b`、多键回退、默认值推断、隐式兼容映射。
- 字段缺失或来源不明确时，必须先停下确认，未确认不得继续编码。
- 涉及接口参数变更时，必须先给出参数映射表：`字段名 -> 来源`，再实施代码修改。
- 若 diff 中出现未约定字段名，视为不符合要求，必须按约定重做。

## 3. 交互与样式硬约束

- 禁止任何点击、按压态位移、缩放、抖动类动画。
- 禁止在共享样式、公共组件、页面局部中定义会导致点击反馈位移或缩放的 `:active` `transform` 或点击触发动画。
- 发现旧的点击摇晃规则时，直接删除原规则，不允许靠页面覆盖兜底。
- 外层功能入口不允许点击后出现选中态高亮、文字变大、字重突变或挤成换行。
- 返回按钮、更多按钮、顶部左右导航按钮不允许任何按压态位移、缩放、抖动或背景跳变。
- 俄罗斯方块时钟页必须保证时间预览清晰可见，不能只生成极暗的轮廓占位。
