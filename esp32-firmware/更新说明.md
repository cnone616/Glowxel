# ESP32 固件更新说明

## 最新更新 - 马里奥表盘

### 新增模式
- `MODE_MARIO` - 马里奥主题表盘，每分钟马里奥跳跃

### 切换模式
WebSocket 命令：
```json
{"cmd": "set_mode", "mode": "clock"}   // 闹钟
{"cmd": "set_mode", "mode": "canvas"}  // 画板
```

### 烧录
```bash
cd esp32-firmware
python3 -m platformio run --target upload
```

## 闹钟自定义功能

### 1. 新增闹钟配置结构 (ClockConfig)

```cpp
struct ClockConfig {
  struct {
    int fontSize;      // 字体大小 1-3
    int x, y;          // 位置坐标
    uint8_t r, g, b;   // RGB 颜色
  } time;
  
  struct {
    bool show;         // 是否显示
    int fontSize;      // 字体大小 1-2
    int x, y;          // 位置坐标
    uint8_t r, g, b;   // RGB 颜色
  } date;
  
  struct {
    bool show;         // 是否显示
    int x, y;          // 位置坐标
    uint8_t r, g, b;   // RGB 颜色
  } week;
};
```

### 2. 默认配置

```cpp
ClockConfig clockConfig = {
  .time = {1, 17, 18, 100, 200, 255},        // 青色时间
  .date = {true, 1, 3, 30, 120, 120, 120},   // 灰色日期
  .week = {true, 23, 44, 100, 100, 100}      // 深灰星期
};
```

### 3. 新增 WebSocket 命令: `set_clock_config`

接收来自 uni-app 的闹钟配置，格式：

```json
{
  "cmd": "set_clock_config",
  "config": {
    "time": {
      "fontSize": 1,
      "x": 17,
      "y": 18,
      "color": {"r": 100, "g": 200, "b": 255}
    },
    "date": {
      "show": true,
      "fontSize": 1,
      "x": 3,
      "y": 30,
      "color": {"r": 120, "g": 120, "b": 120}
    },
    "week": {
      "show": true,
      "x": 23,
      "y": 44,
      "color": {"r": 100, "g": 100, "b": 100}
    }
  }
}
```

### 4. 配置持久化

- 使用 `Preferences` 库保存配置到 Flash
- 命名空间: `"clock"`
- 键名: `"config"`
- 启动时自动加载保存的配置
- 断电后配置不丢失

### 5. 更新 displayClock() 函数

- 根据 `clockConfig` 动态渲染时钟
- 支持自定义字体大小、位置、颜色
- 支持隐藏日期和星期

## 如何更新固件

```bash
cd esp32-firmware
python3 -m platformio run --target upload
```

如遇端口占用：
```bash
lsof | grep cu.usbserial
kill <PID>
```

1. 上传固件到 ESP32
2. ESP32 启动后会显示默认闹钟样式
3. 打开 uni-app 小程序
4. 连接到 ESP32
5. 进入"闹钟编辑器"
6. 修改样式并发送到设备
7. 观察 LED 矩阵显示变化
8. 重启 ESP32，确认配置已保存

## 注意事项

1. 确保 ESP32 已连接 WiFi 并同步了 NTP 时间
2. 配置保存在 Flash 中，频繁写入可能影响 Flash 寿命
3. 如果配置损坏，删除后会自动使用默认配置
4. 字体大小和位置要确保不超出 64x64 显示范围

## 调试信息

固件会在串口输出以下信息：

```
2. 加载闹钟配置...
已加载保存的闹钟配置
```

或

```
2. 加载闹钟配置...
使用默认闹钟配置
```

## 清除配置

如果需要清除保存的闹钟配置，可以在代码中添加：

```cpp
preferences.begin("clock", false);
preferences.clear();
preferences.end();
```

或者通过 WebSocket 发送清除命令（需要自行实现）。
