# ESP32 RenLight 开发环境说明

## 一、环境配置

### 已安装工具

- Python3、PlatformIO（`pio` 命令可用）

### 编译和烧录

```bash
cd esp32-firmware
python3 -m platformio run --target upload
```

### 串口监视（手动执行）

```bash
python3 -m platformio device monitor   # Ctrl+C 退出
```

### 常见问题

- 端口被占用：`lsof | grep cu.usbserial` → `kill <PID>` → 重新烧录
- 检测到蓝牙端口：换一根能传数据的 USB 线，确保 `ls /dev/cu.usb*` 有输出

## 二、设备模式（DeviceMode）

固件当前有 4 个模式，定义在 `display_manager.h`：

```cpp
enum DeviceMode {
  MODE_CLOCK,        // 静态时钟背景模式
  MODE_ANIMATION,    // 动态壁纸模式
  MODE_CANVAS,       // 画板模式
  MODE_TRANSFERRING  // 动画传输临时模式
};
```

### 模式说明

| 模式               | 枚举值 | 前端名称        | 说明                       |
| ------------------ | ------ | --------------- | -------------------------- |
| MODE_CLOCK         | 0      | `clock`         | 静态背景 + 时钟            |
| MODE_ANIMATION     | 1      | `animation`     | 动态背景 + 时钟            |
| MODE_CANVAS        | 2      | `canvas`        | 纯画板实时绘制             |
| MODE_TRANSFERRING  | 3      | `transferring`  | 动画传输中的临时显示状态   |

### 模式切换

- WebSocket 命令 `set_mode` 切换，支持 `"clock"` / `"canvas"` / `"animation"`
- `"clock"` / `"canvas"` / `"animation"` 都是独立模式
- 模式会持久化到 Preferences（`clock` namespace 的 `dispMode` 字段），重启后恢复

## 三、数据存储架构

### 3.1 Preferences（NVS）命名空间

| Namespace | 用途         | 存储内容                            |
| --------- | ------------ | ----------------------------------- |
| `clock`   | 静态时钟配置 | ClockConfig、显示模式、静态背景像素 |
| `anim`    | 动态时钟配置 | ClockConfig、动态背景像素           |

### 3.2 ClockConfig 结构体

```cpp
struct ClockConfig {
  struct { int fontSize; int x; int y; uint8_t r, g, b; } time;
  struct { bool show; int fontSize; int x; int y; uint8_t r, g, b; } date;
  struct { bool show; int x; int y; uint8_t r, g, b; } week;
  struct { bool show; int x; int y; int width; int height; } image;
};
```

### 3.3 存储字段详情

#### `clock` Namespace（静态时钟）

| Key          | 类型  | 说明                                  |
| ------------ | ----- | ------------------------------------- |
| `cfgVer`     | int   | 配置版本号（当前 CONFIG_VERSION = 8） |
| `config`     | bytes | ClockConfig 结构体（静态时钟配置）    |
| `dispMode`   | int   | 当前显示模式（0=CANVAS, 1=ANIMATION） |
| `pixelCount` | int   | 静态背景像素数量                      |
| `pixels`     | bytes | 静态背景像素数据（PixelData 数组）    |

#### `anim` Namespace（动态时钟）

| Key          | 类型  | 说明                               |
| ------------ | ----- | ---------------------------------- |
| `config`     | bytes | ClockConfig 结构体（动态时钟配置） |
| `pixelCount` | int   | 动态背景像素数量                   |
| `pixels`     | bytes | 动态背景像素数据（PixelData 数组） |

### 3.4 像素数据结构

```cpp
struct PixelData {
  uint8_t x;    // 0-63
  uint8_t y;    // 0-63
  uint8_t r, g, b;  // RGB 颜色
};
// 每个像素 5 字节
```

### 3.5 内存中的数据变量

```
ConfigManager::clockConfig          → 静态时钟配置
ConfigManager::animClockConfig      → 动态时钟配置
ConfigManager::staticImagePixels    → 静态背景像素数据（堆内存）
ConfigManager::staticImagePixelCount
ConfigManager::animImagePixels      → 动态背景像素数据（堆内存）
ConfigManager::animImagePixelCount

DisplayManager::canvasBuffer[64][64][3]  → 画板模式的 64x64 RGB 缓冲
DisplayManager::canvasInitialized        → 画布是否已初始化
DisplayManager::blackPixels              → 黑色像素记录（用于时钟区域擦除）
DisplayManager::blackPixelCount
```

### 3.6 GIF 动画存储

GIF 动画数据存储在 LittleFS 文件系统中（`/anim.bin`），由 AnimationManager 管理。
重启后自动从 LittleFS 恢复动画数据。

## 四、WebSocket 通信协议

连接地址：`ws://<设备IP>/ws`

连接成功后设备返回：

```json
{ "status": "connected", "device": "RenLight", "mode": "clock|canvas|animation|transferring" }
```

### 4.1 JSON 命令（文本帧）

| 命令                | 格式                                                                             | 说明                      |
| ------------------- | -------------------------------------------------------------------------------- | ------------------------- |
| `ping`              | `{ "cmd": "ping" }`                                                              | 心跳检测                  |
| `status`            | `{ "cmd": "status" }`                                                            | 获取设备状态              |
| `set_mode`          | `{ "cmd": "set_mode", "mode": "clock\|canvas\|animation" }`                      | 切换模式                  |
| `brightness`        | `{ "cmd": "brightness", "value": 0-255 }`                                        | 设置亮度                  |
| `clear`             | `{ "cmd": "clear" }`                                                             | 清屏                      |
| `pixel`             | `{ "cmd": "pixel", "x": n, "y": n, "r": n, "g": n, "b": n }`                     | 设置单个像素              |
| `set_clock_config`  | 见下方                                                                           | 设置时钟配置              |
| `set_gif_animation` | 见下方                                                                           | 设置 GIF 动画（紧凑格式） |
| `animation_begin`   | `{ "cmd": "animation_begin", "frameCount": n }`                                  | 开始逐帧传输              |
| `animation_frame`   | `{ "cmd": "animation_frame", "index": n, "frame": {...} }`                       | 发送单帧                  |
| `frame_begin`       | `{ "cmd": "frame_begin", "index": n, "type": 1, "delay": ms, "totalPixels": n }` | 分片传输初始化帧          |
| `frame_chunk`       | `{ "cmd": "frame_chunk", "index": n, "pixels": [...] }`                          | 分片传输像素数据          |
| `animation_end`     | `{ "cmd": "animation_end" }`                                                     | 完成动画加载并播放        |

#### set_clock_config 详细格式

```json
{
  "cmd": "set_clock_config",
  "clockMode": "clock|animation",
  "config": {
    "time": {
      "fontSize": 2,
      "x": 22,
      "y": 4,
      "color": { "r": 255, "g": 255, "b": 255 }
    },
    "date": {
      "show": true,
      "fontSize": 1,
      "x": 22,
      "y": 10,
      "color": { "r": 120, "g": 120, "b": 120 }
    },
    "week": {
      "show": true,
      "x": 26,
      "y": 47,
      "color": { "r": 100, "g": 100, "b": 100 }
    },
    "image": { "show": true, "x": 0, "y": 0, "width": 64, "height": 64 }
  }
}
```

- `clockMode` 决定写入 `clockConfig`（静态）还是 `animClockConfig`（动态）

### 4.2 二进制数据（Binary 帧）

像素批量传输使用二进制格式，每像素 5 字节：

```
[x, y, r, g, b] [x, y, r, g, b] ...
```

#### 二进制数据路由逻辑

```
收到二进制数据
  ├─ AnimationManager::receivingFrameIndex >= 0 → 追加到动画帧
  ├─ MODE_CANVAS → 写入 canvasBuffer[64][64][3]（画板实时绘制）
  └─ 其他 → 追加到 imagePixels（背景图，根据模式选择 static/anim）
```

- 画板模式：直接写入 `canvasBuffer` 并实时显示
- 背景图模式：追加到 `imagePixels`，500ms 无新数据后在 `loop()` 中持久化保存

### 4.3 前端发送建议

- 每批 64 像素（320 字节）
- 批间隔 200ms
- 支持分片重组（大 JSON 自动分片）

## 五、显示逻辑

### 5.1 displayClock() 流程

```
1. 根据当前模式选择 ClockConfig 和 imagePixels
2. 获取当前时间
3. 绘制静态背景并缓存背景像素
4. 只恢复时钟文字对应脏区
5. 重绘时间 / 日期 / 星期前景
```

### 5.2 动画模式流程

```
1. AnimationManager 在 loop() 中按帧延迟调度
2. DisplayManager 负责渲染动画背景和时钟前景
3. 首尾都是 `full` 帧时，循环处使用过渡混合，降低硬切感
```

### 5.3 画板模式（实时绘制）

```
1. 前端发送二进制像素数据
2. 写入 canvasBuffer[x][y] = {r, g, b}
3. 立即在 LED 面板上显示
4. canvasBuffer 在断开连接后保持，重连可继续编辑
```

## 六、数据隔离架构图

```
静态时钟模式 (MODE_CANVAS / "clock")
  ├─ 配置: ConfigManager::clockConfig        → Preferences "clock" / "config"
  ├─ 像素: ConfigManager::staticImagePixels   → Preferences "clock" / "pixels"
  └─ 画板: DisplayManager::canvasBuffer       → 内存（不持久化）

动态时钟模式 (MODE_ANIMATION / "animation")
  ├─ 配置: ConfigManager::animClockConfig     → Preferences "anim" / "config"
  ├─ 像素: ConfigManager::animImagePixels     → Preferences "anim" / "pixels"
  └─ GIF:  AnimationManager::currentGIF       → LittleFS "/gif_anim.bin"
```

两种模式的配置和像素数据完全独立，切换模式不会丢失数据。
