MATRIX 像素拼豆设计工具
项目整体技术文档
一、项目背景

MATRIX 是一款面向拼豆爱好者的像素设计工具，支持在移动端进行像素绘图、缩放拖拽、颜色高亮、逐行辅助制作，并可结合实体拼豆板进行实际制作。

本项目目标：

提供专业级拼豆设计能力

优化手机端触控绘图体验

支持实体板制作辅助

后期可扩展为硬件辅助系统

二、系统整体架构

系统分为两部分：

1️⃣ 软件系统（核心）

基于 uni-app + Vue3

多端支持（App + 微信小程序）

核心组件：PixelCanvas.vue

使用 canvas 2d API 渲染像素

架构分层：

UI层
↓
状态管理层
↓
Canvas 渲染层
↓
手势控制层

2️⃣ 硬件系统（规划 / 扩展方向）

未来可扩展：

LED 拼豆辅助板

蓝牙控制模块

行列扫描电路

MCU 控制芯片

手机蓝牙通信控制

手机 → 蓝牙 → MCU → LED矩阵

三、软件系统设计
1️⃣ 技术栈

框架：uni-app

Vue版本：Vue 3

渲染方式：canvas type="2d"

小程序开启：canvas2dApi

manifest.json 核心配置：

"mp-weixin": {
  "setting": {
    "canvas2dApi": true
  }
}
2️⃣ 核心组件 PixelCanvas

功能：

像素绘制

缩放

拖拽

双指缩放

单指绘制

网格绘制

高亮颜色

高亮行

深色模式

关键 props：

参数	作用
width	逻辑像素宽
height	逻辑像素高
zoom	缩放倍数
offsetX	横向偏移
offsetY	纵向偏移
pixels	像素数据 Map
3️⃣ 像素数据结构

使用：

Map<string, string>

key:

"x,y"

value:

"#ff0000"

示例：

"12,8" -> "#000000"

优点：

稀疏存储

高效遍历

内存占用低

适合大尺寸画布

4️⃣ Canvas 渲染机制
原理

逻辑像素：

width = 52
height = 52

实际渲染：

canvas.width = width * zoom
canvas.height = height * zoom

绘制：

ctx.fillRect(x * zoom, y * zoom, zoom, zoom)
5️⃣ 手势控制逻辑

支持：

单指绘制

单指拖动（可切换）

双指缩放

双指平移

算法核心：

记录触点中心

计算两指距离

距离变化 = 缩放

中心点移动 = 平移

四、硬件系统设计（扩展）

未来硬件版本设计目标：

1️⃣ LED 辅助拼豆板

功能：

高亮当前行

高亮当前颜色

2️⃣ MCU 控制系统

建议方案：

结构：

手机 → BLE → MCU → LED矩阵

3️⃣ 行列扫描设计

假设 52×52：

需要 2704 点

采用行列扫描降低 IO 占用

使用驱动芯片（如 MAX7219）

五、核心算法设计
1️⃣ 缩放算法
newZoom = oldZoom + delta

限制范围：

minZoom = 2
maxZoom = 50
2️⃣ 坐标转换公式

触点 → 像素坐标：

pixelX = floor((clientX - rect.left - offsetX) / zoom)
pixelY = floor((clientY - rect.top - offsetY) / zoom)
3️⃣ 高亮算法

两种模式：

颜色高亮

不匹配颜色 → alpha 0.25

匹配 → shadowBlur 10

行高亮

非当前行 → alpha 0.25

当前行 → 正常显示

六、技术难点
1️⃣ 小程序 Canvas 2d 差异

问题：

默认 canvas 300×150

必须手动设置 width / height

dpr 处理

小程序 node 模式异步获取

2️⃣ 性能优化

优化方式：

稀疏 Map 存储

zoom > 1.5 才画网格

避免重复绘制

使用 requestAnimationFrame（后期可优化）

3️⃣ 手势冲突

难点：

单指绘制 vs 单指拖动

双指缩放和平移的协调

防止抖动

七、当前问题

小程序宽高比例不一致

canvas 原生尺寸和逻辑尺寸未完全对齐

高分屏 dpr 处理不完善

大尺寸板绘制时可能性能下降
